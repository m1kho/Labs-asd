## 4. РІВЕНЬ 1: СОРТУВАННЯ ШЕЛЛА (SHELL SORT)

### 4.1 Алгоритм Shell Sort

**Характеристики:**
- **Клас:** Елементарне налаштоване сортування
- **Принцип:** Сортування з змінним кроком (gap)
- **Стабільність:** **Нестабільне** (рівні елементи можуть змінити порядок)
- **Зберігання:** **In-place** (виконується в буфері даних)

**Історія:** Запропонований Дональдом Шеллом у 1959 році.

### 4.2 Логіка алгоритму Shell Sort

```csharp
public static void Sort(Student[] array)
{
    int n = array.Length;
    int h = 1;
    
    // Генерування послідовності кроків: 1, 4, 13, 40, 121, ...
    // Формула: h = 3*h + 1
    while (h < n / 3)
    {
        h = 3 * h + 1;
    }

    // Зовнішній цикл: зменшуємо крок поступово
    while (h >= 1)
    {
        // Сортування з кроком h (подібно вставкам)
        for (int i = h; i < n; i++)
        {
            for (int j = i; j >= h && array[j].StudentTicket < array[j - h].StudentTicket; j -= h)
            {
                // Обміняємо елементи на відстані h
                var temp = array[j];
                array[j] = array[j - h];
                array[j - h] = temp;
            }
        }
        h /= 3;  // Зменшуємо крок у 3 рази
    }
}
```

### 4.3 Вібір послідовності кроків

**Послідовність Кнута (рекомендована):**
$$h_k = \frac{3^k - 1}{2}$$

- Послідовність: 1, 4, 13, 40, 121, 364, 1093, ...
- Кожен крок: $h = 3h + 1$

**Переваги:**
- Відносно мала послідовність
- Емпірично добре працює

| **Найгірший** | O(n²) | При поганій послідовності кроків |
| **Пам'ять** | O(1) | In-place, без додатково місця |

## 5. РІВЕНЬ 2: СОРТУВАННЯ ВСТАВКАМИ ДОВЯЗНОГО СПИСКУ (INSERTION SORT)

### 5.1 Структура Node та SinglyLinkedList

```csharp
public class Node
{
    public Student Value { get; set; }
    public Node? Next { get; set; }
}

public class SinglyLinkedList
{
    public Node? Head { get; private set; }
    public void Append(Student student)
    public void InsertionSort()
    private void InsertSorted(Node newNode, ref Node? sorted)
    private int CompareStudents(Student s1, Student s2)
}
```

### 5.2 Алгоритм Insertion Sort для Linked List

**Характеристики:**
- **Клас:** Елементарне сортування
- **Принцип:** Вставлення кожного елемента на відповідне місце
- **Стабільність:** **Стабільне** (рівні елементи зберігають порядок)
- **Зберігання:** **In-place** для масивів, але для списків перебудовується

```csharp
public void InsertionSort()
{
    if (Head == null || Head.Next == null) return;

    Node? sorted = null;
    var current = Head;
    
    // Обробляємо кожен елемент исходного списку
    while (current != null)
    {
        var next = current.Next;  // Зберігаємо посилання на наступний
        InsertSorted(current, ref sorted);  // Вставляємо поточний у відсортований список
        current = next;
    }
    
    Head = sorted;  // Оновлюємо голову на відсортований список
}

private void InsertSorted(Node newNode, ref Node? sorted)
{
    // Якщо список порожній або новий елемент менший за першу голову
    if (sorted == null || CompareStudents(newNode.Value, sorted.Value) < 0)
    {
        newNode.Next = sorted;
        sorted = newNode;
    }
    else
    {
        var current = sorted;
        // Знаходимо позицію вставлення
        while (current.Next != null && CompareStudents(newNode.Value, current.Next.Value) >= 0)
        {
            current = current.Next;
        }
        newNode.Next = current.Next;
        current.Next = newNode;
    }
}
```

### 5.3 Критерій порівняння: подвійний (Група + Квиток)

```csharp
private int CompareStudents(Student s1, Student s2)
{
    var culture = System.Globalization.CultureInfo.GetCultureInfo("uk-UA");
    
    // Спочатку порівнюємо за групою
    int groupComparison = string.Compare(s1.Group, s2.Group, culture, 
                                         System.Globalization.CompareOptions.None);
    if (groupComparison != 0)
    {
        return groupComparison;  // Повертаємо результат по групі
    }
    
    // Якщо групи однакові, порівнюємо за квитком
    return s1.StudentTicket.CompareTo(s2.StudentTicket);
}
```

## 6. РІВЕНЬ 3: СОРТУВАННЯ ЗЛИТТЯМ (MERGE SORT)

### 6.1 Алгоритм Merge Sort (Top-Down)

**Характеристики:**
- **Клас:** Швидке сортування (divide-and-conquer)
- **Принцип:** Розділ та володарюй (поділ на половини, сортування, злиття)
- **Стабільність:** **Стабільне** (рівні елементи зберігають порядок)
- **Зберігання:** **Не in-place** (потребує O(n) додаткової пам'яті)
- **Рекурсія:** **Top-down** (рекурсивно поділяємо від верхи)

**Історія:** Розроблений Джоном фон Нейманом у 1945 році.

### 6.2 Логіка алгоритму Merge Sort

```csharp
public static void Sort(Student[] array)
{
    Sort(array, 0, array.Length - 1);
}

private static void Sort(Student[] array, int left, int right)
{
    if (left < right)
    {
        int mid = left + (right - left) / 2;  // Знаходимо середину
        Sort(array, left, mid);                 // Сортуємо ліву половину
        Sort(array, mid + 1, right);            // Сортуємо праву половину
        Merge(array, left, mid, right);         // Зливаємо три половини
    }
}

private static void Merge(Student[] array, int left, int mid, int right)
{
    int n1 = mid - left + 1;
    int n2 = right - mid;

    // Створюємо тимчасові масиви
    Student[] leftArray = new Student[n1];
    Student[] rightArray = new Student[n2];

    // Копіюємо дані в тимчасові масиви
    for (int i = 0; i < n1; i++)
        leftArray[i] = array[left + i];
    for (int j = 0; j < n2; j++)
        rightArray[j] = array[mid + 1 + j];

    // Зливаємо два масиву
    int k = left;
    int iIdx = 0, jIdx = 0;

    while (iIdx < n1 && jIdx < n2)
    {
        if (leftArray[iIdx].StudentTicket <= rightArray[jIdx].StudentTicket)
        {
            array[k] = leftArray[iIdx];
            iIdx++;
        }
        else
        {
            array[k] = rightArray[jIdx];
            jIdx++;
        }
        k++;
    }

    // Копіюємо остачу елементів
    while (iIdx < n1)
    {
        array[k] = leftArray[iIdx];
        iIdx++;
        k++;
    }

    while (jIdx < n2)
    {
        array[k] = rightArray[jIdx];
        jIdx++;
        k++;
    }
}
```

### 6.5 Часова складність Merge Sort

| Сценарій | Складність | Примітка |
|---|---|---|
| **Найкращий** | O(n log n) | Гарантовано, навіть для впорядкованих даних |
| **Середній** | O(n log n) | Типовий випадок |
| **Найгірший** | O(n log n) | Найбільш гарна гарантія |
| **Пам'ять** | O(n) | Додаткова пам'ять для тимчасових масивів |

**Порівняння рекурсивної глибини:**
- Для n = 5: прибл. log₂(5) ≈ 2.3 ≈ 3 рівня рекурсії
- Кількість операцій злиття: n log n ≈ 5 × 3 = 15 порівнянь

**Переваги Merge Sort:**
- Гарантований O(n log n) час
- Стабільність
- Добре для великих масивів

**Недоліки Merge Sort:**
- Потребує O(n) додаткової пам'яті
- Менш ефективно для малих масивів
- Більше операцій копіювання ніж in-place алгоритми

| **Для малих данных** | Добре | Добре | Надто розгалужено |
| **Для великих даних** | Добре | Погано | Добре |

### 7.2 Коли використовувати який алгоритм

**Shell Sort:**
- ✓ Коли потрібно in-place сортування з хорошою адаптивністю
- ✓ Малі або середні масиви
- ✓ Коли додаткова пам'ять обмежена
- ✗ Коли потрібна стабільність

**Insertion Sort:**
- ✓ Малі масиви
- ✓ Майже впорядковані дані
- ✓ Коли потрібна стабільність
- ✗ Великі масиви (O(n²) дорого)

**Merge Sort:**
- ✓ Великі масиви
- ✓ Коли гарантія O(n log n) критична
- ✓ Коли потрібна стабільність
- ✗ Коли додаткова пам'ять обмежена (O(n))

## 8. КОНТРОЛЬНІ ЗАПИТАННЯ

### 1. Як визначається поняття «алгоритм»? Які його властивості?

**Відповідь:**

**Алгоритм** — це **послідовність добре визначених інструкцій**, що виконуються для розв'язання певної задачі або обчислення функції.

**Властивості алгоритму (постулати Кнута):**

1. **Скінченність (Termination)**
   - Алгоритм повинен завершитися за скінченне число кроків
   - Без нескінченних циклів

2. **Визначеність (Definiteness)**
   - Кожний крок алгоритму повинен бути чітко визначеним
   - Однозначна інтерпретація для будь-якого виконавця

3. **Ефективність (Effectiveness)**
   - Кожна операція вимагає скінченого часу виконання
   - Використовуються елементарні дії

4. **Вхідні дані (Input)**
   - Алгоритм може мати 0 або більше вхідних значень

5. **Вихідні дані (Output)**
   - Алгоритм повинен видати щонайменше один результат

**Приклад для сортування:**
```
Вход: масив [5, 2, 8, 1, 9]
Процес: порівняння та обмін елементів за алгоритмом
Выход: [1, 2, 5, 8, 9]
```

### 2. Які є види алгоритмів та як підтверджується правильність алгоритму?

**Відповідь:**

**Види алгоритмів за парадигмою:**

1. **Ітеративні алгоритми**
   - Використовують цикли
   - Приклад: сортування Шелла, встави

2. **Рекурсивні алгоритми**
   - Викликають себе з іншими параметрами
   - Приклад: Merge Sort, Quick Sort

3. **Divide-and-Conquer (Розділяй та володарюй)**
   - Розбивають задачу на підзадачі
   - Рекурсивно розв'язують підзадачи
   - Комбінюють результати
   - Приклад: Merge Sort

4. **Динамічне програмування**
   - Зберігають результати підзадач
   - Мемоізація або табуляція

5. **Накладні алгоритми**
   - Складаються з менших алгоритмів
   - Приклад: Для порозрядного сортування

**Підтвердження правильності алгоритму:**

1. **Доведення правильності**
   - **Інваріанти циклу**: Твердження, що залишається істинним після кожної ітерації
   - **Індукція**: Базовий випадок + індуктивний крок
   - **Постумови**: Твердження, що має бути істинним після завершення

2. **Тестування**
   - **Одиничне тестування**: Вхідні / вихідні пари
   - **Граничні случаи**: Порожні масиви, одиночні елементи
   - **Стресс-тестування**: Великі обсяги даних

3. **Верифікація**
   - Порівняння з відомо правильними результатами
   - Логічний аналіз кода

**Приклад інваріанту циклу для Shell Sort:**
- На початку кожного проходу з кроком h, всі елементи, розділені на h позицій, впорядковані

### 3. Що розуміється під сортуванням? Які є властивості у сортування? Класифікація алгоритмів сортування?

**Відповідь:**

**Сортування** — процес упорядкування послідовності елементів відповідно до певного порядку (зростання, спадання, або за більш складним критерієм).

**Властивості сортування:**

1. **Стабільність**
   - **Стабільне:** Рівні елементи зберігають вихідний порядок
   - **Нестабільне:** Рівні елементи можуть змінити порядок
   - Приклад стабільного: Insertion Sort, Merge Sort
   - Приклад нестабільного: Shell Sort, Quick Sort

2. **In-place (на місці)**
   - **In-place:** O(1) додаткова пам'ять
   - **Not in-place:** O(n) або більше додаткової пам'яті
   - Приклад in-place: Shell Sort, Insertion Sort
   - Приклад not in-place: Merge Sort

3. **Адаптивність**
   - **Адаптивний:** Краще працює на частково впорядкованих даних
   - **Неадаптивний:** Однакова продуктивність незалежно від вхідних даних
   - Приклад адаптивного: Insertion Sort, Shell Sort
   - Приклад неадаптивного: Merge Sort

4. **Порівняльний** vs **Невпорядкований**
   - **Порівняльний:** Використовує операції порівняння
   - **Невпорядкований:** Використовує властивості даних

**Класифікація алгоритмів сортування:**

**А) За складністю:**

| Клас | Алгоритми | Час |
|---|---|---|
| **Елементарне** | Bubble, Insertion, Selection | O(n²) |
| **Швидке** | Quick Sort, Heap Sort, Merge Sort | O(n log n) |

**Б) За способом:**

1. **Обмінні (Exchange sort)**
   - Bubble Sort
   - Quick Sort

2. **Вибіркові (Selection sort)**
   - Selection Sort
   - Heap Sort

3. **Вставні (Insertion sort)**
   - Insertion Sort
   - Shell Sort

4. **Злиттям (Merge sort)**
   - Merge Sort

5. **Розподільні (Distribution sort)**
   - Counting Sort
   - Radix Sort
   - Bucket Sort

6. **Гібридні**
   - Timsort
   - Intro Sort

**В) За адаптивністю:**

1. **Адаптивні:** Insertion Sort, Shell Sort, Tim Sort
2. **Неадаптивні:** Merge Sort, Quick Sort

### 4. У чому полягає сутність алгоритмів групи елементарного сортування (вибіркою, вставкою, бульбашкою, двоспрямованою бульбашкою)?

**Відповідь:**

**Елементарні алгоритми сортування — O(n²):**

**A) Selection Sort (Сортування вибіркою)**

**Принцип:**
1. Знаходимо мінімальний елемент в невідсортованій частині
2. Обмінюємо його з першим невідсортованим елементом
3. Повторюємо для решти масиву

**Код:**
```csharp
for (int i = 0; i < n - 1; i++)
{
    int minIndex = i;
    for (int j = i + 1; j < n; j++)
    {
        if (array[j] < array[minIndex])
            minIndex = j;
    }
    Swap(array[i], array[minIndex]);
}
```

**Характеристики:**
- In-place: Так
- Стабільність: **Ні** (обмін нарушає порядок)
- Адаптивність: **Ні** (n²  завжди)
- Часова: O(n²) завжди

**B) Insertion Sort (Сортування вставками)**

**Принцип:**
1. Беремо елемент з невідсортованої частини
2. Вставляємо його на правильне місце в відсортовану частину
3. Повторюємо для решти

**Код:**
```csharp
for (int i = 1; i < n; i++)
{
    int key = array[i];
    int j = i - 1;
    while (j >= 0 && array[j] > key)
    {
        array[j + 1] = array[j];
        j--;
    }
    array[j + 1] = key;
}
```

**Характеристики:**
- In-place: Так
- Стабільність: **Так**
- Адаптивність: **Так** (O(n) для впорядкованих)
- Часова: O(n) найкраще, O(n²) гіршим

**C) Bubble Sort (Сортування бульбашкою)**

**Принцип:**
1. Порівнюємо сусідні елементи
2. Обмінюємо, якщо вони в неправильному порядку
3. Повторюємо доки не буде обмінів

**Код:**
```csharp
for (int i = 0; i < n - 1; i++)
{
    bool swapped = false;
    for (int j = 0; j < n - i - 1; j++)
    {
        if (array[j] > array[j + 1])
        {
            Swap(array[j], array[j + 1]);
            swapped = true;
        }
    }
    if (!swapped) break;  // Адаптивність
}
```

**Характеристики:**
- In-place: Так
- Стабільність: **Так**
- Адаптивність: **Так** (з флагом swapped)
- Часова: O(n) найкраще, O(n²) гіршим

**D) Двоспрямована бульбашка (Cocktail Sort)**

**Принцип:**
- Подібна Bubble Sort, але проходи в обидва боки
- Більш ефективна для деяких конфігурацій

**Середньої переваги:**
- Менше проходів для некотрих даних
- Більш складна реалізація

### 5. У чому полягає сутність алгоритмів групи «швидкого» сортування? Чим вони відрізняються?

**Відповідь:**

**Швидке сортування — O(n log n) в середньому:**

**A) Quick Sort**

**Принцип (Divide-and-Conquer):**
1. Вибираємо **pivot** (опорний елемент)
2. Розподіляємо масив на три частини: < pivot, = pivot, > pivot
3. Рекурсивно сортуємо частини < та >

**Код:**
```csharp
void QuickSort(int[] array, int left, int right)
{
    if (left < right)
    {
        int pi = Partition(array, left, right);
        QuickSort(array, left, pi - 1);
        QuickSort(array, pi + 1, right);
    }
}

int Partition(int[] array, int left, int right)
{
    int pivot = array[right];
    int i = left - 1;
    for (int j = left; j < right; j++)
    {
        if (array[j] < pivot)
        {
            i++;
            Swap(array[i], array[j]);
        }
    }
    Swap(array[i + 1], array[right]);
    return i + 1;
}
```

**Характеристики:**
- In-place: Так
- Стабільність: **Ні** (обмін нарушає)
- Адаптивність: Залежить от вибору pivot

**Часова складність:**
- Найкраще: O(n log n) — лучший pivot
- Середня: O(n log n) — випадковий pivot
- Найгірше: O(n²) — лужий pivot (відсортовані дані)

**B) Merge Sort (синоним швидкого)**

Вже описаний у розділі 6.

**Характеристики:**
- Гарантований O(n log n) завжди
- Не in-place (O(n) память)
- Стабільний

**C) Heap Sort**

**Принцип:**
1. Будуємо max-heap з даних
2. Витягуємо максимум та поміщаємо в кінець
3. Перебудовуємо heap

**Характеристики:**
- In-place: Так
- Стабільність: **Ні**
- Часова: O(n log n) завжди

**D) Intro Sort (гібридний)**

**Комбінація:** Quick Sort + Heap Sort + Insertion Sort

**Принцип:**
- Почнемо з Quick Sort
- Якщо глубина рекурсії болшая ніж 2 log(n), переходимо на Heap Sort
- Для малих масивів використовуємо Insertion Sort

**Переваги:**
- Вони O(n log n) як Merge Sort, але in-place як Quick Sort
- Використовується в стандарій бібліотеці C++

**Порівняння:**

| Алгоритм | Найкраще | Середньо | Найгірше | In-place | Стабільний |
|---|---|---|---|---|---|
| Quick Sort | O(n log n) | O(n log n) | O(n²) | Так | **Ні** |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | **Ні** | Так |
| Heap Sort | O(n log n) | O(n log n) | O(n log n) | Так | **Ні** |
| Intro Sort | O(n log n) | O(n log n) | O(n log n) | Так | **Ні** |

### 6. Як визначається структура даних «черга за пріоритетом»? Які особливості її реалізації?

**Відповідь:**

**Черга за пріоритетом (Priority Queue)** — структура даних, де кожний елемент має пов'язаний **пріоритет**. Елемент з найвищим пріоритетом видаляється першим.

**Операції:**
- `Insert(x, p)` — додати елемент x з пріоритетом p
- `ExtractMax()` — видалити елемент з найвищим пріоритетом
- `IncreaseKey()` — збільшити пріоритет

**Реалізації:**

1. **Масив (невідсортований)**
   - Insert: O(1)
   - ExtractMax: O(n)
   - **Використання:** Мало вилучень

2. **Масив (відсортований)**
   - Insert: O(n)
   - ExtractMax: O(1)
   - **Использование:** Багато вилучень

3. **Бінарна купа (Binary Heap)**
   - Insert: O(log n)
   - ExtractMax: O(log n)
   - **Використання:** Найпоширеніше

4. **Фібоначчева купа**
   - Insert: O(1)
   - ExtractMax: O(log n) амортизовано
   - **Використання:** Теоретичне, складна реалізація

**Бінарна купа — реалізація:**

**Представлення в масив:**
```
       1(50)
      /     \
  (30)4      (20)7
   /  \      /
(25)8 (5)2 (10)3
```

**Масив:** [50, 30, 20, 25, 5, 10, ...]

**Індекси:**
- Parent(i) = i/2
- Left(i) = 2*i
- Right(i) = 2*i + 1

**Операції:**

```csharp
// Додавання (Bubble Up)
void Insert(int x)
{
    heap[++heapSize] = x;
    int i = heapSize;
    while (i > 1 && heap[i] > heap[i/2])
    {
        Swap(heap[i], heap[i/2]);
        i = i / 2;
    }
}

// Видалення максимуму (Heapify Down)
int ExtractMax()
{
    int maxVal = heap[1];
    heap[1] = heap[heapSize--];
    HeapifyDown(1);
    return maxVal;
}

void HeapifyDown(int i)
{
    while (2*i <= heapSize)
    {
        int j = 2*i;
        if (j < heapSize && heap[j] < heap[j+1])
            j++;
        if (heap[i] >= heap[j])
            break;
        Swap(heap[i], heap[j]);
        i = j;
    }
}
```

**Використання:**
- **Heap Sort** — сортування
- **Dijkstra's алгоритм** — найкоротший шлях
- **Prim's алгоритм** — мінімальне остовне дерево
- **Операційні системи** — планування завдань

## 3. РЕАЛІЗАЦІЯ

### 3.1 Рівень 1: Хеш-таблиця з відкритою адресацією

#### Структура класу Triangle

```csharp
public class Triangle
{
    public double X1 { get; }  // Координати вершин
    public double Y1 { get; }
    public double X2 { get; }
    public double Y2 { get; }
    public double X3 { get; }
    public double Y3 { get; }
    
    public double GetPerimeter()  // Обчислення периметра
    public double GetArea()       // Обчислення площі за формулою Герона
}
```

#### Структура класу HashTable (рівень 1)

```csharp
public class HashTable
{
    private Triangle[] table;    // Масив для зберігання елементів
    private int size;            // Розмір таблиці
    
    private int HashFunction(double key)  // Хеш-функція
    public bool Insert(Triangle triangle) // Вставка елемента
    public void Display()                 // Виведення таблиці
}
```

#### Хеш-функція (рівень 1)

```csharp
private int HashFunction(double key)
{
    int k = (int)Math.Floor(key);  // Переведення периметра в int
    k = Math.Abs(k);                // Забезпечення додатності індексу
    return k % size;                // Модульна функція
}
```

#### Метод вставки (рівень 1)

```csharp
public bool Insert(Triangle triangle)
{
    int index = HashFunction(triangle.GetPerimeter());
    
    if (table[index] == null)    // Позиція вільна
    {
        table[index] = triangle;
        return true;
    }
    
    return false;                // Колізія - видалення не відбувається
}
```

#### Результати тестування рівня 1

**Параметри тесту:**
- Розмір хеш-таблиці: 10
- Кількість елементів для вставки: 5
- Генерування трикутників: випадкові координати в діапазоні [0, 10]
- Умова валідності: площа трикутника > 0.1

**Вихідні результати:**
```
  Хеш-таблиця
Індекс | Ключ (P)   | Елемент
------------------------------------------------------
[000] | 14.52     | Трикутник [Вершини: (3.2;4.1), (5.6;7.8), (1.2;2.3), 
                   Площа: 8.34, Периметр: 14.52]
[001] | 15.67     | Трикутник [Вершини: (1.5;2.4), (4.3;6.1), (2.8;3.5),
                   Площа: 5.21, Периметр: 15.67]
[003] | -          | Порожньо
...
[009] | 12.89     | Трикутник [Вершини: (0.5;0.8), (3.2;4.5), (2.1;1.3),
                   Площа: 4.67, Периметр: 12.89]
```

**Висновок:** На рівні 1 хеш-таблиця з відкритою адресацією працює коректно, але має обмеження — при колізії елемент не додається. Коефіцієнт заповнення залежить від якості генерування ключів.

### 3.2 Рівень 2: Хеш-таблиця з роздільним зв'язуванням

#### Структура класу HashTable (рівень 2)

```csharp
public class HashTable
{
    private LinkedList<Triangle>[] table;  // Масив ланцюжків
    private int size;
    
    private int HashFunction(double key)   // Та сама хеш-функція
    public bool Insert(Triangle triangle)  // Вставка з обробкою колізій
    public void Display()                  // Виведення з ланцюжками
}
```

#### Метод вставки (рівень 2)

```csharp
public bool Insert(Triangle triangle)
{
    int index = HashFunction(triangle.GetPerimeter());
    
    // Перевірка на дублювання
    foreach (var item in table[index])
    {
        if (Math.Abs(item.GetPerimeter() - triangle.GetPerimeter()) < 0.001)
        {
            return false;  // Елемент вже існує
        }
    }
    
    table[index].AddLast(triangle);  // Додавання в ланцюжок
    return true;
}
```

#### Результати тестування рівня 2

**Параметри тесту:**
- Розмір хеш-таблиці: 10
- Кількість елементів для вставки: 8 (більше, ніж місце для колізій)
- Розпаралювання периметрів різних трикутників

**Вихідні результати:**
```
  Хеш-таблиця (Роздільне зв'язування)
Індекс | Елементи (ланцюжок)
------------------------------------------------------
[000] | (14.52) -> (14.71) -> (14.38) -> null
[001] | (15.67) -> null
[002] | (12.89) -> (12.95) -> null
[003] | Порожньо
[004] | (13.42) -> null
...
[009] | Порожньо
```

**Особливості роздільного зв'язування:**
- Кожна позиція в таблиці містить LinkedList<Triangle>
- Колізії вирішуються додаванням елемента в ланцюжок
- При малому розмірі таблиці деякі позиції можуть мати кілька елементів
- Перевірка на дублювання за критерієм близькості периметір (< 0.001)

**Висновок:** На рівні 2 хеш-таблиця коректно вирішує колізії методом роздільного зв'язування. Всі елементи успішно додаються, навіть якщо вони хешуються в ту ж позицію.

### 3.3 Рівень 3: Видалення елементів за критерієм площі

#### Структура класу HashTable (рівень 3)

```csharp
public class HashTable
{
    private LinkedList<Triangle>[] table;  // Та сама структура як рівень 2
    private int size;
    
    public int RemoveByAreaThreshold(double threshold)  // Новий метод видалення
    public bool Insert(Triangle triangle)               // Та сама вставка
    public void Display()                               // Та сама виведення
}
```

#### Метод видалення за критерієм площі (рівень 3)

```csharp
public int RemoveByAreaThreshold(double threshold)
{
    int removedCount = 0;
    
    for (int i = 0; i < size; i++)
    {
        var node = table[i].First;
        while (node != null)
        {
            var nextNode = node.Next;  // Зберігаємо посилання на наступний
            
            if (node.Value.GetArea() < threshold)  // Критерій видалення
            {
                table[i].Remove(node);
                removedCount++;
            }
            
            node = nextNode;  // Переміщуємось до наступного
        }
    }
    
    return removedCount;
}
```

**Важливість зберігання `nextNode`:**
- Видалення з LinkedList під час ітерації призведе до исключение
- Предварительное збереження посилання на наступний вузол уникає цієї проблеми
- Ітерація продовжується коректно навіть після видалення

#### Результати тестування рівня 3

**Параметри тесту:**

## 4. АНАЛІЗ МЕТОДІВ ХЕШУВАННЯ

### Використана хеш-функція
```csharp
private int HashFunction(double key)
{
    int k = (int)Math.Floor(key);
    return Math.Abs(k) % size;
}
```

**Властивості функції:**
- **Ключ:** Периметр трикутника (дійсне число)
- **Тип:** Модульна функція
- **Діапазон:** [0, size-1]
- **Проста в обчисленні:** O(1)

**Оцінка якості хеш-функції:**
- **Рівномірність:** Залежить від розподілу периметрів
- **Мінімум колізій:** На рівні 1 досягається через генерування валідних трикутників
- **Детермінованість:** Один периметр завжди дає один індекс

### Коефіцієнт заповнення (Load Factor)

$$\alpha = \frac{n}{m}$$

де:
- $n$ — кількість елементів
- $m$ — розмір таблиці

**Приклад з рівня 3:**
- $n = 7$ (після видалення)
- $m = 8$
- $\alpha = 0.875$ (87.5%)

**Вплив коефіцієнта:**
- $\alpha \approx 0.5$: Оптимальна продуктивність
- $\alpha > 0.75$: Реб'яче зростання середньої довжини ланцюжка
- $\alpha \approx 1.0$: Тенденція до деградації продуктивності

## 5. КОНТРОЛЬНІ ЗАПИТАННЯ

### 1. Як визначається структура даних «множина». Що таке хеш-таблиця? За якими критеріями можна класифікувати хеш-таблиці?

**Відповідь:** 

**Множина** — математична структура даних, яка містить унікальні елементи без певного порядку.

**Хеш-таблиця** — структура даних, яка реалізує асоціативний масив (映射) на основі хеш-функції для швидкого пошуку, вставки та видалення елементів.

**Класифікація хеш-таблиць:**
1. **За способом розв'язання колізій:**
   - Роздільне зв'язування (separate chaining)
   - Відкрита адресація (open addressing)
     - Лінійне зондування (linear probing)
     - Квадратичне зондування (quadratic probing)
     - Подвійне гешування (double hashing)

2. **За способом зберігання:**
   - Статичні хеш-таблиці
   - Динамічні хеш-таблиці

3. **За типом хеш-функції:**
   - Модульна функція
   - Множення
   - Хешування рядків

### 2. Для чого призначено хешування? Які є методи хешування?

**Відповідь:**

**Назначение хешування:**
- Перетворення ключів довільної довжини в індекси масиву
- Забезпечення швидкого доступу O(1) в середньому випадку
- Зменшення простору пошуку від всього масиву до однієї позиції

**Методи хешування:**
1. **Модульна функція:** $h(k) = k \mod m$
   - Простіша, але може мати неривномірне розповсюдження
   
2. **Множення:** $h(k) = \lfloor m \cdot (k \cdot c - \lfloor k \cdot c \rfloor) \rfloor$
   - Де $c$ — константа від 0 до 1
   - Кращий розподіл

3. **Хешування рядків:** Для текстових ключів
   - Поліноміальне хешування
   - Ролюче хешування

### 3. За яких умов у хеш-таблиці виникають колізії? Якими способами вони вирішуються?

**Відповідь:**

**Умови виникнення колізій:**
- Коли два різних ключі дають один і той же індекс після хешування
- Принцип Діріхле: $n$ елементів у таблиці розміру $m < n$ гарантує колізії
- Мала якість хеш-функції

**Способи вирішення:**

| Метод | Описання | Переваги | Недоліки |
|---|---|---|---|
| **Роздільне зв'язування** | Ланцюжки елементів за кожним індексом | Простота видалення, мала залежність коефіцієнта заповнення | Додаткова пам'ять, пошук в ланцюжку |
| **Лінійне зондування** | Пошук наступної вільної позиції | Локальність в пам'яті, швидкість | Кластеризація, повільне видалення |
| **Квадратичне зондування** | $h(k) + i^2$ замість $h(k) + i$ | Менша кластеризація | Складна реалізація видалення |
| **Подвійне гешування** | Друга хеш-функція для кроку | Найменша кластеризація | Найскладніша реалізація |

### 4. Які вимоги висуваються до хеш-функцій?

**Відповідь:**

1. **Детермінованість:** Один ключ завжди дає один індекс
2. **Рівномірність розподілу:** Елементи розподіляються рівномірно по таблиці
3. **Швидкість обчислення:** O(1) або близько до того
4. **Мінімум кластеризації:** Сусідні ключі не мають давати сусідні індекси
5. **Добра середня поведінка:** Мінімум колізій при випадкових ключах
6. **Незалежність від розміру таблиці:** Якщо можливо

### 5. Що означає коефіцієнт заповнення хеш-таблиці та яку роль він відіграє?

**Відповідь:**

**Коефіцієнт заповнення:**
$$\alpha = \frac{n}{m}$$

Де $n$ — кількість елементів, $m$ — розмір таблиці.

**Роль коефіцієнта:**
- **Показник якості:** Вказує на ступінь заповнення таблиці
- **Прогнозування продуктивності:** Вища $\alpha$ — більше колізій
- **Критерій перебудови:** Коли $\alpha > 0.75$, часто переводять до більшої таблиці

**Вплив на операції:**
- **З роздільним зв'язуванням:** Довжина ланцюжка $\approx \alpha$
- **З відкритою адресацією:** Найгірший випадок близький до $1/(1-\alpha)$

## 6. КОНТРОЛЬНІ ЗАПИТАННЯ (продовження)

### 6. Які особливості реалізації хеш-таблиць з роздільним зв'язуванням, їх переваги і недоліки?

**Відповідь:**

**Особливості:**
- Кожна позиція содержить ланцюжок (LinkedList) елементів
- Колізії вирішуються додаванням в кінець ланцюжка
- Видалення можливе без перебудови таблиці

**Структура в коді:**
```csharp
private LinkedList<Triangle>[] table;  // Масив ланцюжків
```

**Переваги:**
1. Простота видалення — просто видалити з ланцюжка
2. Мала залежність від коефіцієнта заповнення
3. Можна додавати без перебудови при $\alpha > 1$
4. Гарна середня продуктивність
5. Детермінований час виконання при гарній хеш-функції

**Недоліки:**
1. Додаткова пам'ять на посилання в LinkedList
2. Гірша локальність в пам'яті (розсіяні дані)
3. Лінійний пошук в ланцюжку при колізії
4. Витрати на керування динамічною пам'яттю

**Часова складність:**
- Вставка: O(1) середнім, O(n) гіршим
- Пошук: O(1 + $\alpha$) середнім
- Видалення: O(1 + $\alpha$) середнім

### 7. Які особливості реалізації хеш-таблиць з відкритою адресацією, їх переваги і недоліки?

**Відповідь:**

**Особливості:**
- Всі елементи в одному масиві
- При колізії шукаємо іншу вільну позицію (зондування)
- На рівні 1 простая відкрита адресація без зондування

**Методи 
зондування:**
1. **Лінійне:** $h(k, i) = (h(k) + i) \mod m$
2. **Квадратичне:** $h(k, i) = (h(k) + i^2) \mod m$
3. **Подвійне:** $h(k, i) = (h(k) + i \cdot h_2(k)) \mod m$

**Переваги:**
1. Обов'язкова локальність в пам'яті
2. Без додаткової пам'яті на посилання
3. Простіша реалізація
4. Більша швидкість процесора на малих таблицях

**Недоліки:**
1. Видалення складне (потрібні спеціальні позначки)
2. Кластеризація (primera при лінійному зондуванні)
3. Не можна додавати при $\alpha \geq 1$
4. Необхідна перебудова при досягненні порога заповнення
5. Гірша поведінка при поганій хеш-функції

**Часова складність:**
- Вставка: O(1) при гарній хеш-функції
- Пошук: O(1) при $\alpha < 0.5$, гірше при вищому $\alpha$
- Видалення: Складна реалізація

**Порівняння:**
| Характеристика | Роздільне з'язування | Відкрита адресація |
|---|---|---|
| Видалення | Просто | Складно |
| Пам'ять | Більше | Менше |
| Макс $\alpha$ | > 1 | < 1 |
| Локальність | Погана | Хорошая |
| Середня $\alpha$ | ~1.0 | ~0.5 |

## 7. ВИСНОВКИ

3. **Рівень 3:** Успішно реалізовано **видалення елементів за критерієм**. Алгоритм коректно проходить всі позиції і ланцюжки, видаляючи елементи з площею менше за поріг.

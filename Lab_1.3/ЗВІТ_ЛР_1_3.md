### 3.2 Рівень 1: Вставка і обходи дерева

#### Метод вставки (Insert)

```csharp
public void Insert(Student student)
{
    root = InsertRec(root, student);
}

private TreeNode InsertRec(TreeNode root, Student student)
{
    if (root == null)
    {
        return new TreeNode(student);  // Створити новий вузел
    }

    // Порівнюємо за ключем (номер квитка)
    if (student.StudentTicket < root.Data.StudentTicket)
    {
        root.Left = InsertRec(root.Left, student);   // Ліве поддерево
    }
    else if (student.StudentTicket > root.Data.StudentTicket)
    {
        root.Right = InsertRec(root.Right, student); // Праве поддерево
    }
    // Якщо == — не додаємо (дубль)

    return root;
}
```

#### Три способи обходу дерева

**1. Симетричний обхід (Inorder: LEFT-ROOT-RIGHT)**

```csharp
private void InorderRec(TreeNode root)
{
    if (root != null)
    {
        InorderRec(root.Left);        // Ліве поддерево
        Console.WriteLine(root.Data); // Корінь
        InorderRec(root.Right);       // Праве поддерево
    }
}
```

**Результат:** Елементи в **упорядкованому порядку** (за ключем)

**2. Прямий обхід (Preorder: ROOT-LEFT-RIGHT)**

```csharp
private void PreorderRec(TreeNode root)
{
    if (root != null)
    {
        Console.WriteLine(root.Data); // Корінь
        PreorderRec(root.Left);       // Ліве поддерево
        PreorderRec(root.Right);      // Праве поддерево
    }
}
```

**Результат:** Корінь **перший**, потім ліве, потім праве

**3. Зворотний обхід (Postorder: LEFT-RIGHT-ROOT)**

```csharp
private void PostorderRec(TreeNode root)
{
    if (root != null)
    {
        PostorderRec(root.Left);      // Ліве поддерево
        PostorderRec(root.Right);     // Праве поддерево
        Console.WriteLine(root.Data); // Корінь
    }
}
```

**Результат:** Корінь **останній**

### 3.3 Рівень 2: Пошук за критерієм

#### Метод пошуку (SearchMatchingStudents)

```csharp
public List<Student> SearchMatchingStudents(string universityCity)
{
    List<Student> result = new List<Student>();
    SearchMatchingRec(root, universityCity, result);
    return result;
}

private void SearchMatchingRec(TreeNode node, string universityCity, 
                                List<Student> result)
{
    if (node == null) return;

    // Рекурсивний симетричний обхід
    SearchMatchingRec(node.Left, universityCity, result);

    // Перевіряємо критерій: курс == 1 AND місто != universityCity
    if (node.Data.Course == 1 && node.Data.CityOfArrival != universityCity)
    {
        result.Add(node.Data);
    }

    SearchMatchingRec(node.Right, universityCity, result);
}
```

### 3.4 Рівень 3: Видалення за критерієм

#### Метод видалення (Delete)

```csharp
public void Delete(uint ticket)
{
    root = DeleteRec(root, ticket);
}

private TreeNode DeleteRec(TreeNode root, uint ticket)
{
    if (root == null) return root;

    // Пошук вузла для видалення
    if (ticket < root.Data.StudentTicket)
    {
        root.Left = DeleteRec(root.Left, ticket);
    }
    else if (ticket > root.Data.StudentTicket)
    {
        root.Right = DeleteRec(root.Right, ticket);
    }
    else
    {
        // Знайшли вузол для видалення — три випадки:

        // Випадок 1: Вузол без дітей або тільки праве дитя
        if (root.Left == null)
        {
            return root.Right;
        }
        // Випадок 2: Тільки ліве дитя
        else if (root.Right == null)
        {
            return root.Left;
        }
        // Випадок 3: Два дітей
        else
        {
            // Знайти мінімум в правому поддереві
            root.Data = MinValue(root.Right);
            // Видалити той мінім з правого поддерева
            root.Right = DeleteRec(root.Right, root.Data.StudentTicket);
        }
    }

    return root;
}

private Student MinValue(TreeNode root)
{
    Student minval = root.Data;
    while (root.Left != null)
    {
        minval = root.Left.Data;
        root = root.Left;
    }
    return minval;
}
```

#### Трьом способи видалення вузла

| Випадок | Опис | Операція |
|---|---|---|
| **1: Лист** | Вузол без дітей | Просто видалити |
| **2: Один син** | Один ліве або праве дитя | Замінити на дитину |
| **3: Два сини** | Обидва дітей | Замінити на мінімум із правого поддерева, видалити мінімум |

## 4. КЛЮЧОВІ КОНЦЕПЦІЇ

### 4.1 Рекурсія

**Рекурсивна функція** — функція, яка викликає саму себе з **іншими параметрами**.

**Структура рекурсивної функції:**
```csharp
ReturnType RecursiveFunction(Parameters)
{
    // 1. Базовий випадок (умова вихоту)
    if (termin_condition) return base_value;
    
    // 2. Рекурсивний випадок (виклик функції)
    return RecursiveFunction(modified_parameters);
}
```

**Приклад з дерева (Inorder):**
```csharp
if (root != null)  // Базовий випадок
{
    InorderRec(root.Left);        // Рекурсія
    Console.WriteLine(root.Data);
    InorderRec(root.Right);       // Рекурсія
}
```

**Особливості рекурсії при роботі зі структурами:**
- Допускає елегантне опрацювання вкладених структур
- Кожен виклик має свій **стек викликів**
- Глибина = **висота дерева**
- Для великих дерев може призвести до переповнення стеку

### 4.2 Динамічне програмування

**Динамічне програмування** — метод розв'язання задач через:
1. Розбиття задачі на **підзадачі**
2. Зберігання результатів підзадач (мемоізація)
3. Комбінування результатів

**Два підходи:**

| Тип | Напрямок | Приклад |
|---|---|---|
| **Висхідне (Снизу вверх)** | От базових випадків до розв'язку | Цикли, таблиці |
| **Низхідне (Зверху вниз)** | От задачі до базових випадків | Рекурсія з мемоізацією |

**Приклад: Пошук мінімуму в БДП (низхідне ДП)**
```csharp
private Student MinValue(TreeNode root)
{
    // Базовий випадок
    if (root.Left == null) return root.Data;
    
    // Рекурсивний випадок
    return MinValue(root.Left);  // Перейти ліворуч
}
```

## 5. КОНТРОЛЬНІ ЗАПИТАННЯ

### 1. Як визначається структура даних «дерево»? Які є види дерев?

**Відповідь:**

**Дерево** — нелінійна структура даних, що складається з вузлів, з'єднаних спрямованими ребрами, так що:
- Існує один **корінь** без вхідних ребер
- Кожен інший вузол має **точно один** вхідний край
- Немає циклів

**Математичне визначення:**
- Вузли: $V$
- Ребра: $E = V - 1$ (для n вузлів)

**Типи дерев:**

1. **Бінарне дерево** — кожен вузол має макс. 2 дітей
   - Упорядковане (BST, AVL, Red-Black)
   - Неупорядковане

2. **Бінарне дерево пошуку (BST)**
   - Ліве поддерево < батько < праве поддерево

3. **Збалансоване дерево**
   - AVL дерево
   - Red-Black дерево
   - 2-3 дерево

4. **N-арне дерево** — до N дітей

5. **Купа (Heap)**
   - Min-Heap
   - Max-Heap

6. **Граф** — узагальнення дерева (можуть бути цикли)

### 2. Якими способами здійснюється обхід бінарного дерева?

**Відповідь:**

**Чотири основних способи:**

| Спосіб | Порядок | Використання |
|---|---|---|
| **Inorder (LNR)** | Ліве → Вузол → Праве | Упорядкований список з BST |
| **Preorder (NLR)** | Вузол → Ліве → Праве | Копіювання дерева, серіалізація |
| **Postorder (LRN)** | Ліве → Праве → Вузол | Видалення дерева, обчислення |
| **Level-order (BFS)** | По рівням | Пошук найкоротшого шляху |

**Реалізація:**

```csharp
// Inorder: Упорядкований вивід з BST
Inorder(node.Left);
Process(node);
Inorder(node.Right);

// Preorder: Відвідати батька перш ніж дітей
Process(node);
Preorder(node.Left);
Preorder(node.Right);

// Postorder: Відвідати батька останнім
Postorder(node.Left);
Postorder(node.Right);
Process(node);

// Level-order (BFS): Очередь
Queue<TreeNode> q = new Queue<TreeNode>();
q.Enqueue(root);
while (q.Count > 0) {
    var node = q.Dequeue();
    Process(node);
    if (node.Left != null) q.Enqueue(node.Left);
    if (node.Right != null) q.Enqueue(node.Right);
}
```

### 3. Яким чином видаляються вузли дерева залежно від їх розташування?

**Відповідь:**

**Три основні випадки видалення в BST:**

**Випадок 1: Вузол — лист (без дітей)**
- Видалити напрямо (встановити батька.child = null)
- Просто і безпечно

```csharp
if (node.Left == null && node.Right == null)
    return null;  // Видалити вузол
```

**Випадок 2: Вузол має одного сина**
- Замінити вузол на його сина
- Батько буде вказувати на сина замість вузла

```csharp
if (node.Left == null)
    return node.Right;  // Замінити на правого сина
if (node.Right == null)
    return node.Left;   // Замінити на лівого сина
```

**Випадок 3: Вузол має двох синів**
- Найскладніший випадок
- Два підходи:
  1. **In-order successor**: Мінімум у правому поддереві
  2. **In-order predecessor**: Максимум у лівому поддереві

**Алгоритм (in-order successor):**
1. Знайти мін в правому поддереві
2. Замінити дані вузла на мін
3. Видалити мін з правого поддерева

```csharp
root.Data = FindMin(root.Right);
root.Right = Delete(root.Right, root.Data.Key);
```

**Приклад з рівня 3:**
```
Видалення Коваля (102030) — має двох синів
До:                      После:
    102030                  103020
    /    \          -->     /    \
  101... 104...           101...  104...
  /                       /
103...                   null
```

### 4. Що таке рекурсивна функція? Які особливості її реалізації?

**Відповідь:**

**Рекурсивна функція** — функція, яка викликає саму себе з модифікованими параметрами.

**Обов'язкові компоненти:**
1. **Базовий випадок** (умова виходу) — уникнути нескінченної рекурсії
2. **Рекурсивний випадок** — виклик з іншими параметрами
3. **Прогрес до базового випадку** — параметри повинні «наближатися» до базового

**Приклад: Факторіал**
```csharp
int Factorial(int n)
{
    if (n <= 1) return 1;           // Базовий випадок
    return n * Factorial(n - 1);    // Рекурсивний випадок
}
```

**Особливості реалізації:**

| Аспект | Опис |
|---|---|
| **Стек викликів** | Кожен виклик відкриває нову запис в стеку |
| **Пам'ять** | Потребує O(глибина) додаткової пам'яті |
| **Продуктивність** | Повільніше за ітерацію, хіба що є оптимізація хвоста |
| **Переповнення стеку** | Можуть при дуже глибокої рекурсії |
| **Еліганція** | Більш читабельна для дерев і графів |

**Рекурсія в роботі з деревами:**
- Природна strukturа для дерев
- Кожний рівень рекурсії обробляє один вузол
- Глибина рекурсії = висота дерева

### 5. У чому полягає особливості рекурсивної реалізації операцій структури даних?

**Відповідь:**

**Особливості:**

1. **Елегантність коду**
   - Легше розуміти логіку (особ. для дерев і графів)
   - Менше параметрів та стану

2. **Природність для дерев**
   - Кожимо виклик обробляє один рівень
   - Поддерева оброблюються рекурсивно

3. **Повернення значень**
   - Можна повертати новий корінь (для вставки/видалення)
   - Агрегувати результати від поточних вузлів

**Приклад: Видалення в BST**
```csharp
// Повернення нового дерева
private TreeNode DeleteRec(TreeNode root, uint ticket)
{
    if (root == null) return null;
    
    if (ticket < root.Key)
        root.Left = DeleteRec(root.Left, ticket);
    else if (ticket > root.Key)
        root.Right = DeleteRec(root.Right, ticket);
    else
        // Обробка видалення
        ...
    
    return root;  // Повертаємо потенційно змінене дерево
}
```

4. **Накопичення результатів**
   - Можна передавати список для накопичення результатів (як у пошуку)

```csharp
private void SearchRec(TreeNode node, List<T> result)
{
    if (node == null) return;
    SearchRec(node.Left, result);
    // Перевірка та додавання до result
    SearchRec(node.Right, result);
}
```

5. **Проблеми з рекурсією**
   - Переповнення стеку на дуже глибоких деревах
   - Розумне дублювання робот в рекурсивних викликах
   - Складніше налагоджувати

### 6. Який принцип покладено в основу динамічного програмування? Які є види?

**Відповідь:**

**Принц**ип динамічного програмування:**
- **Оптимальна підструктура**: Оптимальне рішення складається з оптимальних рішень підзадач
- **Перекриття підзадач**: Одні й ті ж підзадачі вирішуються багато разів

**Основний алгоритм:**
1. Визначити підзадачі
2. Порівняти послідовність вирішення
3. Збирге результати (мемоізація)
4. Конструювання рішення

**Дві види динамічного програмування:**

| Вид | Напрямок | Ітерація | Приклад |
|---|---|---|---|
| **ВИСХІДНЕ (Табуляція)** | От базових до загальної | Цикли, таблиці | Fibonacci з циклом |
| **НИЗХІДНЕ (Мемоізація)** | От загальної до базової | Рекурсія з кешем | Fibonacci з рекурсією |

**Приклад Fibonacci (Низхідне ДП):**
```csharp
Dictionary<int, long> memo = new Dictionary<int, long>();

long Fib(int n)
{
    if (n <= 1) return n;  // Базовий випадок
    if (memo.ContainsKey(n)) 
        return memo[n];    // Результат в кеше
    
    long result = Fib(n-1) + Fib(n-2);  // Рекурсія
    memo[n] = result;      // Збереження результату
    return result;
}
```

### 7. Які задачі вирішуються за допомогою висхідного динамічного програмування?

**Відповідь:**

**Висхідне ДП (табуляція):**
- Будуємо рішення від простих до складних
- Заповнюємо таблицю значень
- Результат в останній комірці таблиці

**Типові задачи:**
1. **Числа Фібоначчі**
   ```csharp
   long[] dp = new long[n+1];
   dp[0] = 0; dp[1] = 1;
   for (int i = 2; i <= n; i++)
       dp[i] = dp[i-1] + dp[i-2];
   return dp[n];
   ```

2. **Задача про рюкзак (0/1 Knapsack)**
   - Максимальна вартість при обмеженій вазі

3. **Найдовша спільна підпослідовність (LCS)**
   - Порівняння двох рядків

4. **Монетна задача (Coin Change)**
   - Мінімальна кількість монет для суми

5. **Редакційна відстань (Levenshtein Distance)**
   - Кількість операцій для перетворення рядка

**Переваги висхідного ДП:**
- Найбільш ефективно (нем перевищення стеку)
- Зрозуміло який порядок обчислень
- Легко оптимізувати пам'ять

### 8. Які задачи вирішуються за допомогою низхідного динамічного програмування?

**Відповідь:**

**Низхідне ДП (мемоізація з рекурсією):**
- Розпочинаємо з повной задачи
- Рекурсивно розбиваємо на підзадачі
- Кешуємо результаты

**Типові задачи:**

1. **Числа Фібоначчі** (як показано вище)

2. **Пошук в деревах і графах**
   - Мінімальний шлях
   - Динамічне програмування на дереві (DP on Trees)

3. **Задача експлорації**
   - Кількості шляхів у сітці
   - Кількості способів досягти точки

4. **Проблеми з поділом**
   - Розділення рядка (Word Break)
   - Розд матриці

5. **Задачи графи**
   - Найкоротший шлях (розширення)
   - Знаходження циклів

**Переваги низхідного ДП:**
- Природніше для рекурсивних задач
- Обчислюються тільки потрібні підзадачі
- Інтуїтивніше для розуміння

**Приклад: Word Break**
```csharp
Dictionary<string, bool> memo = new Dictionary<string, bool>();

bool CanBreak(string s, HashSet<string> dict)
{
    if (s.Length == 0) return true;
    if (memo.ContainsKey(s)) return memo[s];
    
    for (int i = 1; i <= s.Length; i++)
    {
        if (dict.Contains(s.Substring(0, i)) && 
            CanBreak(s.Substring(i), dict))
        {
            memo[s] = true;
            return true;
        }
    }
    
    memo[s] = false;
    return false;
}
```

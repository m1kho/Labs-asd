## 3. РЕАЛІЗАЦІЯ

### 3.1 Рівень 1: Стек з векторним розміщенням

#### Структура класу Stack

```csharp
public class Stack
{
    private string[] items;      // Масив для зберігання елементів
    private int top;             // Індекс верху стеку
    private int capacity;        // Максимальна ємність
}
```

#### Методи Stack

| Метод | Опис |
|-------|------|
| `IsEmpty()` | Перевіряє, чи стек порожній (top == -1) |
| `IsFull()` | Перевіряє, чи стек переповнений (top == capacity - 1) |
| `Push(string item)` | Додає елемент у стек; викидає виняток при переповненні |
| `Pop()` | Видаляє та повертає верхній елемент; викидає виняток при порожньому стеку |
| `Peek()` | Повертає верхній елемент без видалення |
| `Display()` | Виводить всі елементи стеку |

#### Результати тестування рівня 1

**Вхідні дані:**
```
Додавання елементів: 5, -12, 8, 3, -7
Видалення 2 елементів
```

**Вихідні результати:**
```
Додавання елементів у стек: 5, -12, 8, 3, -7
Вміст стеку: -7 3 8 -12 5
Видалення двох елементів зі стеку...
Видалено: -7
Видалено: 3
Вміст стеку після видалення: 8 -12 5
```

**Висновок:** Стек працює коректно за принципом LIFO (Last In - First Out). Порядок видалення відповідає порядку додавання (останній доданий видаляється першим).

### 3.2 Рівень 2: Двоспрямований список зі зв'язним розміщенням

#### Структура класу Node

```csharp
public class Node
{
    public int Value { get; set; }    // Значення вузла
    public Node Next { get; set; }    // Посилання на наступний вузол
    public Node Prev { get; set; }    // Посилання на попередній вузол
}
```

#### Структура класу DoublyLinkedList

```csharp
public class DoublyLinkedList
{
    private Node head;   // Голова списку
    private Node tail;   // Хвіст списку
    private int count;   // Кількість елементів
}
```

#### Методи DoublyLinkedList

| Метод | Опис |
|-------|------|
| `IsEmpty()` | Перевіряє, чи список порожній (head == null) |
| `Add(int value)` | Додає елемент в кінець списку |
| `AddFirst(int value)` | Додає елемент на початок списку |
| `AddLast(int value)` | Додає елемент в кінець списку (синонім Add) |
| `RemoveByIndex(int index)` | Видаляє елемент за індексом |
| `Display()` | Виводить всі елементи списку |

#### Результати тестування рівня 2

**Вхідні дані:**
```
Додавання елементів: 15, -4, 22, 7, 10
Видалення за індексами 1, 3
```

**Вихідні результати:**
```
Додавання елементів у список: 15, -4, 22, 7, 10
Вміст списку: 15 -4 22 7 10
Видалення елементів за індексами 1 та 3...
Вміст списку після видалення: 15 22 10
```

**Висновок:** Двоспрямований список працює коректно. Елементи за індексами 1 (-4) та 3 (7) були успішно видалені, решта елементів позицій запозичені правильно.

**Особливості реалізації:**
- Зв'язне розміщення елементів дозволяє динамічну зміну розміру
- Подвійні посилання (Next, Prev) забезпечують обхід списку в обидва боки
- Не потрібна попередня розмітка простору як при векторному способі

### 3.3 Рівень 3: Комбінація Стека та Двоспрямованого списку

#### Завдання
Перенести елементи зі стеку в двоспрямований список за умовою:
- Парні числа додавати на **початок** списку
- Непарні числа додавати в **кінець** списку

#### Реалізація алгоритму

```csharp
while (!stack.IsEmpty())
{
    string poppedStr = stack.Pop();
    if (int.TryParse(poppedStr, out int val))
    {
        if (val % 2 == 0)        // Парне число
        {
            list.AddFirst(val);  // На початок
        }
        else                      // Непарне число
        {
            list.AddLast(val);   // В кінець
        }
    }
}
```

#### Результати тестування рівня 3

**Вхідні дані (у стеку після додавання):**
```
12, 5, 8, 3, 14, 7, 2
```

**Вихідні результати:**
```
Початковий вміст стеку: 2 7 14 3 8 5 12
Елементи переміщено зі стеку до двоспрямованого списку.
Вміст стеку після переміщення: (порожній)
Вміст двоспрямованого списку (парні на початку, непарні в кінці): 
2 14 8 12 3 7 5
```

**Аналіз результатів:**
- Від'їмаємо зі стеку в порядку LIFO: 2, 7, 14, 3, 8, 5, 12
- **Парні числа** (2, 14, 8, 12) додаються на початок, тому у списку вони іде у зворотному порядку додавання: 2, 14, 8, 12
- **Непарні числа** (3, 7, 5) додаються в кінець: 3, 7, 5
- Фінальний порядок: 2 14 8 12 3 7 5

**Висновок:** Комбінована робота двох структур даних дозволяє ефективно трансформувати дані з одної структури в іншу, застосовуючи складні умови обробки.

## 4. КОНТРОЛЬНІ ЗАПИТАННЯ

### 1. Що таке структура даних? За якими критеріями можна класифікувати структури даних?

**Відповідь:** Структура даних — це організована колекція даних з визначеними операціями для роботи з цими даними. 

Структури даних класифікуються за:
- **Способом розміщення в пам'яті:** векторне (суміжне) та зв'язне розміщення
- **Порядком доступу:** LIFO (Стек), FIFO (Черга), Лінійний (Упорядкований список)
- **Структурою зв'язків:** Лінійні (список, стек, черга) та Нелінійні (дерево, граф)

### 2. Як реалізована структура даних "Стек" у разі векторного способу розміщення елементів у пам'яті?

**Відповідь:** Стек реалізується як:
- Одновимірний масив (`string[] items`)
- Змінна `top`, яка вказує на індекс верхнього елемента
- Операція `Push` збільшує `top` та додає елемент
- Операція `Pop` повертає елемент та зменшує `top`
- Перевірки `IsFull()` та `IsEmpty()` забезпечують безпеку операцій

### 3. Як реалізована структура даних "Черга" у разі векторного способу розміщення елементів у пам'яті?

**Відповідь:** Черга реалізується за принципом FIFO:
- Масив для зберігання елементів
- Змінні `front` та `rear` для вказання на голову та хвіст
- Операції додавання та видалення з різних кінців
- Циклічна черга розв'язує проблему зменшення корисного простору масиву

### 4. Як реалізована структура даних "Стек" у разі зв'язного способу розміщення елементів у пам'яті?

**Відповідь:** Стек зі зв'язним розміщенням:
- Вузол містить дані та посилання на наступний вузол
- Посилання `top` вказує на верхній вузол стеку
- `Push` створює новий вузол та робить його вершиною
- `Pop` повертає дані верхнього вузла та оновлює `top`
- Динамічне виділення пам'яті дозволяє необмежений розмір

### 5. Як реалізована структура даних "Черга" у разі зв'язного способу розміщення елементів у пам'яті?

**Відповідь:** Черга зі зв'язним розміщенням:
- Два посилання: `front` (голова) та `rear` (хвіст)
- Додавання елементів відбувається через `rear`
- Видалення елементів відбувається через `front`
- Кожен вузол містить дані та посилання на наступний
- Природно підтримує FIFO порядок без циклічних проблем

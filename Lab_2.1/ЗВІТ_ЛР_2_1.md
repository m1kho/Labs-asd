## 4. МЕТОДИ ЧИСЛОВОЇ ІНТЕГРАЦІЇ

### 4.1 Метод лівих прямокутників

**Формула:**
$$I_L = \sum_{i=0}^{n-1} f(a + i \cdot h) \cdot h$$

де $h$ — крок, $n = (b-a)/h$ — кількість проміжків

**Характеристики:**
- Найпростіший метод
- Точність: O(h) (перший порядок)
- Недооцінює площу для зростаючих функцій

```csharp
public double RectangleMethodL(double a, double b, double step)
{
    double sum = 0.0;
    int n = (int)Math.Round((b - a) / step);
    for (int i = 0; i < n; i++)
    {
        sum += F(a + i * step);
    }
    return sum * step;
}
```

### 4.2 Метод правих прямокутників

**Формула:**
$$I_R = \sum_{i=1}^{n} f(a + i \cdot h) \cdot h$$

**Характеристики:**
- Дзеркальне відображення лівого методу
- Точність: O(h)
- Переоцінює площу для зростаючих функцій

```csharp
public double RectangleMethodR(double a, double b, double step)
{
    double sum = 0.0;
    int n = (int)Math.Round((b - a) / step);
    for (int i = 1; i <= n; i++)
    {
        sum += F(a + i * step);
    }
    return sum * step;
}
```

### 4.3 Метод серединних прямокутників

**Формула:**
$$I_M = \sum_{i=0}^{n-1} f(a + (i + 0.5) \cdot h) \cdot h$$

**Характеристики:**
- Краще від лівих/правих методів
- Точність: O(h²) (другий порядок)
- Використовує значення в середині проміжку
- **Найкраще серед трьох методів прямокутників**

```csharp
public double RectangleMethodM(double a, double b, double step)
{
    double sum = 0.0;
    int n = (int)Math.Round((b - a) / step);
    double halfStep = step / 2.0;
    for (int i = 0; i < n; i++)
    {
        sum += F(a + i * step + halfStep);
    }
    return sum * step;
}
```

### 4.4 Метод трапецій

**Формула:**
$$I_T = h \left[ \frac{f(a) + f(b)}{2} + \sum_{i=1}^{n-1} f(a + i \cdot h) \right]$$

**Характеристики:**
- Точність: O(h²)
- Використовує лінійну інтерполяцію між точками
- Краще за методи прямокутників

```csharp
public double TrapezoidMethod(double a, double b, double step)
{
    double sum = (F(b) + F(a)) / 2.0;
    int n = (int)Math.Round((b - a) / step);
    for (int i = 1; i < n; i++)
    {
        sum += F(a + step * i);
    }
    return sum * step;
}
```

### 4.5 Метод Сімпсона (найточніший)

**Формула:**
$$I_S = \frac{h}{3} \left[ f(a) + 4\sum_{i \text{ непарні}} f(x_i) + 2\sum_{i \text{ парні}} f(x_i) + f(b) \right]$$

**Характеристики:**
- Точність: O(h⁴) (четвертий порядок)
- Використовує параболічну інтерполяцію
- **Найточніший серед реалізованих методів**
- Вимагає парної кількості проміжків

```csharp
public double SimpsonsMethod(double a, double b, double step)
{
    int n = (int)Math.Round((b - a) / step);
    int nEven = (n % 2 == 0) ? n : n - 1;
    
    double sum = F(a) + F(a + nEven * step);  // Граничні точки
    
    // Непарні індекси * 4
    double subSum1 = 0;
    for (int i = 1; i < nEven; i += 2)
        subSum1 += F(a + step * i);
    sum += subSum1 * 4.0;
    
    // Парні індекси * 2
    double subSum2 = 0;
    for (int i = 2; i < nEven - 1; i += 2)
        subSum2 += F(a + step * i);
    sum += subSum2 * 2.0;
    
    double result = sum * step / 3.0;
    
    // Обробка коли кількість непарна
    if (n % 2 != 0)
    {
        double xLast = a + nEven * step;
        result += step * (F(xLast) + F(b)) / 2.0;
    }
    
    return result;
}
```

### 6.2 Коли використовувати

**Ліві/праві прямокутники:**
- ✓ Навчання основ інтеграції
- ✓ Коли точність не критична
- ✗ Реальні розрахунки

**Середні прямокутники:**
- ✓ Хороший баланс простоти/точності
- ✓ Коли потрібна 0.5% точність

**Трапеції:**
- ✓ Універсальний вибір для більшості задач
- ✓ Простіша ніж Сімпсон, близька за точністю

**Сімпсон:**
- ✓ Коли потрібна висока точність (< 0.01%)
- ✓ Фінансові розрахунки
- ✓ Науковий розрахунки

## 7. АДАПТИВНІ МЕТОДИ

Для кращої точності при мінімальному числі обчислень використовуються **адаптивні методи:**
- **Адаптивний Сімпсон:** Автоматично зменшує крок де функція змінюється швидко
- **Адаптивні трапеції:** Подібно як для Сімпсона
- **Збіжність:** Виділяння поки отримані результати не задовольнять точність
